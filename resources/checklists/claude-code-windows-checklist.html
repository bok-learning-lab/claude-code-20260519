Claude Code Setup — Windows

Open PowerShell (Start → "PowerShell"). Run commands one at a time. Estimated time: 15–25 min.

Before you start
Confirm you have a paid Claude account (Pro, Max, Team, or Enterprise)
Open PowerShell and run winget --version
If "not recognized," install App Installer from the Microsoft Store, then reopen PowerShell
Confirm winget --version shows a version number
Step 1 — Install core tools
winget install --id Git.Git -e --accept-source-agreements --accept-package-agreements
winget install --id OpenJS.NodeJS -e --accept-source-agreements --accept-package-agreements
winget install --id Microsoft.VisualStudioCode -e --accept-source-agreements --accept-package-agreements
winget install --id Python.Python.3.13 -e --accept-source-agreements --accept-package-agreements
Close PowerShell completely and open a fresh window
Step 2 — Verify core tools
git --version → shows a version number
node --version → shows a version number
npm --version → shows a version number
If "running scripts is disabled": run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser, type Y, reopen PowerShell
python --version → shows a version number
If it opens the Microsoft Store: Settings → Apps → Advanced app settings → App execution aliases → turn off python.exe and python3.exe
code --version → shows a version number
If "not recognized": close & reopen PowerShell. Still fails? Open VS Code → Ctrl+Shift+P → "Shell Command: Install 'code' command in PATH"
Step 3 — Install Claude Code
irm https://claude.ai/install.ps1 | iex
Close PowerShell completely and open a fresh window
claude --version → shows a version number
If "not recognized," add %USERPROFILE%\.local\bin to PATH — see full guide
claude doctor → no errors
Step 4 — Clone the workshop repo
cd $env:USERPROFILE
mkdir Development; cd Development
git clone https://github.com/bok-learning-lab/claude-code-20260519.git
Step 5 — Open in VS Code
cd claude-code-20260519
code . → VS Code opens with the project
If prompted "Do you trust the authors?" → click Yes, I trust the authors
Step 6 — Log in to Claude
In VS Code: Terminal → New Terminal (or Ctrl+`)
Confirm prompt ends in ...\claude-code-20260519
Run claude → follow login prompts, sign in via browser
Back in terminal, type "What files are in this project?" → Claude responds
Step 7 — VS Code extension (optional)
Ctrl+Shift+X → search Claude Code → install (publisher: Anthropic)
Click the ✱ Spark icon in the Activity Bar → sign in
Send a test message in the panel
